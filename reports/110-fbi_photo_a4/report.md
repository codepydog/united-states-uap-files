# #110 FBI Photo A4：左半邊整片空白

![photo](photo.png)

打開 A4 的瞬間有點奇怪。

左邊將近一半的畫面是純白的，沒有任何 sensor 影像、沒有 HUD、沒有遮蔽長條。看起來像是 PowerPoint 投影片裡某個圖片被放錯位置的感覺。實際的 sensor 截圖只佔據右側大約 2/3 的範圍，畫面比例完全不對稱。

這不是 sensor 本身的輸出。

軍方 FLIR / EO sensor 的原始畫面是規整的長方形（4:3 或 16:9，依平台型號）。A4 這種「左邊大量空白」的呈現方式，幾乎可以肯定是**後製階段的編輯結果**：原本可能存在於畫面左側的某些東西（額外的 HUD 元素、不同的 sensor 視野、文字標註、或是另一個 sensor 的副螢幕），在解密過程中被整片塗白。

對照 A1 至 A3，那三張的遮蔽都用黑色長條，邊界整齊。A4 改用白色大片塗抹，遮蔽手法截然不同。可能的解讀有兩種：

1. **不同的解密處理人員**：A1-A3 用一種規範，A4 由另一位處理時改用另一種規範
2. **遮蔽對象不同**：A1-A3 遮的是 metadata 文字，A4 左半邊原本是別的東西（不只是文字），所以用白色大片覆蓋而不是黑條

第二種解讀更值得想。如果 A4 原本是個**多畫面組合**——例如左邊一個 sensor 視野、右邊另一個 sensor 視野同步呈現——那麼左半邊被整片塗白，意味著官方不希望讓觀眾比對兩個視角。

這種「dual-pane」呈現在軍方 sensor 系統中常見：F-35 的 EOTS + DAS、AC-130 的 MX-20 + MX-25 並排、F/A-18 的 HUD + ATFLIR 同框。對應 D 系列已揭露的案件，[#046 D28 Ayn al Asad](../046-dow_uap_d28_mission_report_ayn_al_asad_iraq_september_2024/report.md) 中 AC-130J Ghostrider「MX-20 + MX-25 雙感測器同時被 UAP 穿越視野」就是這種雙螢幕配置。

A4 是不是同一個雙螢幕配置的截圖、左半邊被整片移除？沒辦法直接證明，但形態上對得起來。

## 右半邊的山地

把目光放到實際看得到的部分。

右側的地面紋路比 A2 和 A3 都要明確——可以看到山脊的稜線、陰影面、以及大致平行的紋路（可能是沙丘的脊線、或是被風吹出來的線條地貌）。整體偏明亮，意味白天拍攝、地表溫度高。

準星的位置不在畫面正中央，而是偏右。這暗示 sensor 操作員當時還在追蹤過程中，目標尚未完全 boresight，或者就是兩個 sensor 視角的其中一個（左邊那個本來會看到目標的另一面）。

目標本身是一個比 A3 更小的暗點，位於準星交點的右下方。形狀仍然偏圓，但因為尺寸比 A3 小，邊緣的解析度比較模糊。如果這是同一個物體在不同瞬間的觀測，那麼**目標相對 sensor 的距離可能變遠了**，或者 sensor 變焦改變了倍率。

## 序列的中段

A1 到 A4 連起來看，能感覺到一個進展又收回的過程：
- A1：遠遠的一個點，背景無特徵
- A2：地面出現，目標看得清楚
- A3：目標進入準星正中央，輪廓最銳利
- A4：目標再次變遠，準星偏離正中央，旁邊還有半個畫面被整片白色蓋掉

如果這真的是按時序的同一場觀測，那麼這個目標在 A3 之後**遠離了 sensor**——可能是它本身移動了、可能是平台轉向、也可能是時間推移後物體的相對方位變化。

或者根本不是同一個觀測。A1 至 A8 雖然編號連續，但 FBI 沒有任何文字說明它們之間的關係。它們可能來自八次完全不同的事件，只是因為性質類似而被歸類在一起。

「5」依然在畫面右側準星附近。但 A4 的「5」字位置略有不同——準星本身偏離畫面中央，「5」的相對位置也跟著偏。可以確認的是：A1-A4 的 sensor 設定（不管那個「5」代表什麼）保持一致。
